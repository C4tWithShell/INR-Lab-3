Answers to questions

How do VLANs work at a packet level?

The IEEE 802.1 Q standard changes the structure of the Ethernet frame, - inserts a tag (4 Byte), which specifies:

  • Tag Protocol ID (2 Bytes);
  • Tag Control Information (2 Bytes) - User Priority, Canonical Format Indicator, VLAN ID.

Input process:
Puts a tag. If the frame has a tag, then it is sent to the forwarding process without changes, if without the tag, then it is placed on the frame according to the input rule.

Forwarding process:
Makes decisions about filtering or forwarding a packet to the destination port according to the VLAN and MAC tables.

Output process:
Determines whether to leave the VLAN attribute in the frame. If it is known that a VLAN-Unauthorized device is connected to the port, then the tag is removed.

What are the two major protocols used for this?

802.1Q and Cisco ISL

What do we mean by Native VLAN?

Native VLAN means the port that defines the VLAN number that all untagged packets receive.

Implementation

Answers to questions

What is Link Aggregation?

Link Aggregation is the integration of several parallel data transmission channels in Ethernet networks into one logical.

How does it work (briefly)?

Adding additional links will lead to loops in the network. With Link Aggregation thread uses only one of the links. Different threads are assigned to different links based on the hashing algorithm, so together you get the full bandwidth of the combined links, but anyone thread will get the bandwidth of only one link.

What are the possible configuration modes?

LAG (without protocol), LACP, and PAgP (Cisco only)

Configuring Link Aggregation

First, I added a new interface on MikroTik with balance-rr mode.
interface bonding add name=bond1 slaves=ether2,ether4 mode=balance-rr
Then I disabled the IP address for ether 2 and set it to the bond1.
ip address add address=192.168.1.1/24 interface=bond1
To apply the configuration, I rebooted MikroTik. Then I set up a second bond to the web.

There is a loop between switches.

This is because when the first switch receives a broadcast ping, it transmits it to the other two. The second one will receive it and send it back to the first one, as well as to the third one. Etc, etc.

This leads to an overload of switches and inactivity of this network segment.

STP is on

Then I turned on STP on the switches and captured the traffic again (Figure 19).


Figure 19 - Captured traffic with STP

Now there is no loop, and also now I can see the STP traffic (Figure 20).


Figure 20 - STP packet

This is the configuration BPDU that the root switch regularly sends to others to build the topology. It contain several fields, let’s focus on the most important - sender ID (Bridge ID), the ID of the root switch (Root Bridge ID), ID of the port from which this packet was sent (Port ID), and the cost of the route to the root switch (Root Path Cost).

I make the internal switch as root by adding the following line to the bridge mstpctl-treeprio 8192 (Default = 32768). The switch with the lower value is root. After that, the line between the administration and the ITdepartment is not used.

Would we need STP between routers?

We don’t need STP between router, but there can be a routing loop between them. Then we use OSPF (Open Shortest Path First).
